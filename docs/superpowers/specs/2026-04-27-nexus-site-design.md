# NEXUS Site -- Design Specification

**Date:** 2026-04-27

---

## 1. Overview

A documentation and community site for NEXUS, built with Astro Starlight, deployed to GitHub Pages. The site serves two audiences: developers who want to build modules or contribute, and end users who want to install and run NEXUS. Developer experience is the priority.

The site auto-generates module and kernel reference documentation from the NEXUS codebase using AST parsing, so docs stay accurate without manual upkeep.

### Build Pipeline

1. `python scripts/generate-docs.py` -- scans `nexus/modules/` and `nexus/kernel/`, writes markdown to `src/content/docs/reference/`
2. `npm run build` (runs `astro build`) -- builds the static site
3. GitHub Actions deploys the `dist/` output to GitHub Pages

### Typography

- **Headings:** JetBrains Mono (monospace, technical, matches README terminal aesthetic)
- **Body:** Inter (readable, clean, technical)
- **Code:** JetBrains Mono

### Visual Effects

- Subtle glow/bloom on accent elements via `box-shadow` with cyan/violet
- Faint dot-matrix or grid background texture on hero section
- Module cards: thin glowing border on hover (`1px solid` -> `0 0 12px` glow)
- Sharp corners throughout -- no border-radius except where functionally needed (inputs, buttons: 4px max)
- Ruled lines, intelligence dashboard aesthetic

### Module Reference (`/docs/reference/modules/`)

Auto-generated. One page per module. Each page contains:
- Module name and tier
- One-line description (from class docstring first line)
- Full docstring as overview
- Cortex routing keywords (from `_MODULE_KEYWORDS`)
- Class signature
- Public methods with signatures and docstrings
- Dataclasses/types defined in the module file

### Kernel Reference (`/docs/reference/kernel/`)

Auto-generated. One page per kernel component. Same structure as module reference: class docstring, public methods, types.

## 5. Doc Generation Script

`site/scripts/generate-docs.py`

### Approach

Uses Python's `ast` module exclusively -- no runtime imports of NEXUS code, no dependency installation required. Parses source files as syntax trees.

### Extraction

For each `.py` file in `nexus/modules/` and `nexus/kernel/` (skipping `__init__.py`, `base.py`):

1. Parse file with `ast.parse()`
2. Find the primary class (first class definition in the file)
3. Extract:
   - Class name
   - Class docstring (`ast.get_docstring()`)
   - `name` and `description` class attributes (if defined as string assignments)
   - `keywords` list attribute (if defined)
   - Public methods (not starting with `_`): name, arguments, return annotation, docstring
   - Dataclasses defined in the same file: name, fields with types and defaults
4. For Cortex specifically: extract `_MODULE_KEYWORDS` dict via AST

### Output

Each extracted module/kernel component produces a markdown file with Starlight frontmatter:

```markdown
---
title: Oracle
description: Anticipatory trigger engine
sidebar:
  order: 1
---

## Overview

{class docstring}

## Routing Keywords

{keywords from Cortex._MODULE_KEYWORDS}

## API

### `class Oracle`

{constructor signature}

### Methods

#### `handle(message, context)`

{method docstring}

...

## Types

### `TriggerRule`

{dataclass fields}
```

### Routing Table

Additionally generates a `routing.md` page that shows the complete keyword-to-module mapping table from `Cortex._MODULE_KEYWORDS`.

## 6. GitHub Actions Deployment

`.github/workflows/deploy-site.yml`

### Triggers

Push to `main` that touches:
- `site/**`
- `nexus/**` (source changes should rebuild reference docs)

### Steps

1. Checkout repository
2. Setup Python 3.11
3. Setup Node 20
4. Cache `node_modules` by `site/package-lock.json` hash
5. Run `python site/scripts/generate-docs.py`
6. `cd site && npm ci && npm run build`
7. Deploy `site/dist/` to GitHub Pages via `actions/deploy-pages@v4`

### Configuration

The Astro config sets `base` and `site` for GitHub Pages:
- `site`: `https://allstreets.github.io`
- `base`: `/NEXUS`

## 7. Constraints

- No light mode
- No JavaScript frameworks beyond Astro (no React, no Vue)
- No external API calls at build time beyond npm registry
- No runtime JavaScript except Starlight's built-in search and the architecture diagram hover tooltips
- Static output only (GitHub Pages compatible)
- The generate-docs.py script must work without installing NEXUS dependencies (AST-only parsing)
